## Running Apps

Start all apps:

```bash
bun run dev
```

Start individually:

```bash
cd apps/gateway
bun run dev
```

## Service Architecture

### What Each Service Does

| Service | Purpose |
|---------|---------|
| **DWS** | Coordinator/marketplace - routes requests to Node providers |
| **Node app** | Provider - does actual work (inference, storage, compute) |
| **Crucible** | Agent orchestrator - runs agent logic, calls DWS for resources |
| **Indexer** | Blockchain indexer - queries on-chain data |

### DWS Service Types

| Type | Description | Hardware Needed |
|------|-------------|-----------------|
| **Inference** | AI model execution (LLMs, image models) | GPU |
| **Storage** | File storage, IPFS pinning | Disk space |
| **Compute** | Serverless functions, containers | CPU/RAM |
| **CDN** | Content caching and delivery | Bandwidth |

### Agent Provisioning Flow

When you create an AI agent:

1. **Define agent** вҶ’ Crucible stores agent config
2. **Agent runs** вҶ’ Crucible executes agent runtime
3. **Agent thinks** вҶ’ Crucible calls DWS вҶ’ DWS routes to Node (inference)
4. **Agent stores data** вҶ’ Crucible calls DWS вҶ’ DWS routes to Node (storage)
5. **Agent executes on-chain** вҶ’ Crucible submits transaction to L2

**Key point**: DWS without Node providers is an empty marketplace. You need at least one Node app providing services for DWS to route requests to.

## Startup Order

Services must be started in dependency order:

1. **L1 Geth** (no dependencies)
2. **PostgreSQL** (no dependencies)
3. **op-geth** (depends on L1)
4. **op-node** (depends on L1, op-geth)
5. **op-batcher** (depends on op-node)
6. **Indexer** (depends on L2, PostgreSQL)
7. **DWS** (depends on L2)
8. **Crucible** (depends on DWS)
9. **Gateway** (depends on L2)
10. **Client apps** (depend on services)
